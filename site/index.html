Unattended BMAD dev-loop
orchestrator.

Point it at a repo and a problem statement. It drives BMAD's own workflows — brainstorm → PRD → UX → architecture → epics/stories → per-story dev loop — through to shipped, goal-verified stories. It switches Claude models per phase and pings your phone only when it genuinely needs a decision.

Why it's different

Every BMAD phase runs as its own fresh Agent SDK session — no session is ever resumed across phases or stories. All cross-session context travels through documents.

🧠

Per-phase model switching

Planning gets a strong model's judgment; bookkeeping gets a cheap one; coding gets a code-tuned model; review gets a skeptical, separate pass.

🧱

Context cleared between stories

Bounded cost regardless of project size. Documents — BMAD artifacts plus a managed tracking block — are the interface between steps, not chat history.

🔒

Fail-closed goal verification

A separate review session diffs against a baseline commit and checks acceptance criteria against the actual code. Unparseable verdict = fail, never a silent ship.

📱

Phone in the loop

Console, desktop, and ntfy fan-out; Telegram is bidirectional. Reply /retry, /skip, or free-text guidance from your phone — no server to run.

💾

Crash-safe & resumable

State persists atomically after every step. chucknorris resume picks up cleanly after a crash or Ctrl-C.

🪓

Caveman output style

Optional terse mode compresses model chatter across the whole run. Code, commits, JSON verdicts, and safety text stay exact — brevity never costs correctness.

Output style

🪓 Caveman mode

Configured once at first run (and overridable per run with --caveman), it appends a terseness instruction to every orchestrated session — cutting the fluff that most session cost is spent on. Source code, commit messages, structured JSON verdicts, and security/irreversible-action text are always exempted.

off — normal prose lite — trim filler full — classic terse ultra — maximally terse

Quickstart

No network, no API spend to validate. Real runs need Claude access.

# install
$ npm install && npm run build && npm link

# first run — one-time global config (notifications, caveman style, models)
$ chucknorris run --target /path/to/your/repo

# verify the notification round-trip before a long unattended run
$ chucknorris notify-test

# resume after a crash or Ctrl-C
$ chucknorris resume --target /path/to/your/repo

# validate the codebase (35+ unit + e2e tests, no network, no spend)
$ npx tsc --noEmit && npm test
